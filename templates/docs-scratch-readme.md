# Scratch Notes

本目錄保存暫時筆記、原始貼文、未整理資料與討論草稿。

## Rules

1. scratch 可以不完整。
2. scratch 可以矛盾。
3. scratch 可以過期。
4. scratch 不可直接作為 implementation source。
5. 若 scratch 內容要進入正式流程，必須先整理到 discovery、requirements、contracts、schemas 或 active spec。

## Directory Organization

本目錄可依實際需要使用子目錄分類，例如依日期、topic、meeting 或 source material 分組。

第一版不預先限制固定分類；但本 README 應維護基本導覽或使用規則，避免暫存資料被誤認為正式來源。Scratch 可以較鬆散，但檔名仍應足以追溯來源或用途。

## Data Safety

不要在 scratch 保存 credential、token、password、private key、真實客戶資料或 production payload。
